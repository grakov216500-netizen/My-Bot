# Интеграция Novel в «Планы и задачи»

Novel — Notion-style WYSIWYG редактор. Интеграция позволяет добавлять и редактировать задачи с форматированием (жирный, списки, заголовки и т.п.).

## Шаг 1. Установка зависимостей и сборка

Откройте терминал в папке `site/plans-editor` и выполните:

```bash
cd site/plans-editor
npm install
npm run build
```

После сборки в папке `site/dist/` появятся файлы:
- `plans-editor.js` — основной скрипт редактора
- `plans-editor.css` — стили (если генерируются)

## Шаг 2. Подключение в index.html

В `site/index.html` перед закрывающим `</body>` добавьте:

```html
<!-- Novel editor для Планов (загружается при открытии раздела) -->
<script type="module" src="dist/plans-editor.js" id="plans-editor-script"></script>
```

Или подключайте динамически при первом открытии «Планы» (уже реализовано в app.js).

## Шаг 3. Как это работает

1. **Добавление задачи** — вместо простого поля ввода открывается редактор Novel с поддержкой:
   - Slash-команд (`/` для заголовков, списков, цитат)
   - Форматирование (жирный, курсив, подчёркивание)
   - Списки, чекбоксы

2. **Отображение** — в списке задач показывается извлечённый текст из JSON (или HTML для старых задач).

3. **Хранение** — контент сохраняется в поле `text` таблицы `tasks` в формате **TipTap JSON** (строка).

## Шаг 4. Структура файлов

```
site/
├── plans-editor/           # React + Novel (сборка)
│   ├── package.json
│   ├── vite.config.js
│   └── src/
│       └── main.jsx
├── dist/                   # Результат npm run build
│   ├── plans-editor.js
│   └── plans-editor.css
├── app.js                  # Интеграция (динамическая загрузка)
└── index.html
```

## Требования

- Node.js 18+
- npm или yarn

## AI-автодополнение (опционально)

Novel поддерживает AI. Для включения нужен API-эндпоинт. По умолчанию отключено (`disableLocalStorage`), редактор работает без AI.
